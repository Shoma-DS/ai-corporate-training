# 演習データ → Google Workspace 作成ワークフロー

ユーザーが「演習データを Google スプレッドシート/ドキュメントにして」「gws で演習用シートを作って」「演習データを Drive に上げて」と言ったときに使う。

## ゴール

1回（セッション）の `演習データ/` から、Google ドライブの「演習データ」フォルダの中に次を作る。

- **CSV → スプレッドシート**: その回の `*.csv` を **1つの Google スプレッドシート**にまとめる。CSV が複数ある場合は、CSV ごとに**シート（タブ）**を追加する。1回につき演習用スプレッドシートは**1つ**にする。
- **ワークシート系 → Google ドキュメント**: `*.md` / `*.txt`（`README.md` を除く）は **1ファイルずつ Google ドキュメント**にする。
- 作成物はすべて、その回の Drive 上の `演習データ` フォルダの中に入れる。My Drive 直下に放置しない。

## 前提

- `gws`（Google Workspace CLI）が認証済みであること（`gws drive files list --params '{"pageSize":1}'` で確認）。
- ファイル作成・Drive 移動は外部送信を伴う。**必ず `--dry-run` で内容を確認し、ユーザー承認を得てから本実行する**。
- 演習データは公開安全なダミーのみ（個人情報・実顧客データ・価格・認証情報を含まない）。`session-specific-exercise-data-workflow.md` のデータ設計ルールに従う。

## 手順

1. 対象セッションの `演習データ/` を確認し、CSV とワークシート（md/txt）を把握する。

   ```bash
   ls '講座/COURSE/01-セッション名/演習データ/'
   ```

2. Drive 側の演習データフォルダを決める。次のどちらか。
   - 既に Drive 上に演習データフォルダがある場合は、その**フォルダ ID** を `--drive-folder-id` で渡す（最優先・最も安全）。
   - 講座/回/演習データ の階層をルートフォルダから自動で辿って作る場合は、起点フォルダの ID を `--root-folder-id` で渡す。`<講座名>/<回名>/演習データ` を順に find-or-create する。

3. まず `--dry-run` で、作られるスプレッドシートのタイトル・タブ構成・ドキュメント名を確認する。

   ```bash
   python3 skills/corporate-training-course-builder/scripts/build_exercise_gws_docs.py \
       --session-dir '講座/COURSE/01-セッション名' \
       --drive-folder-id 1AbCxyz... \
       --dry-run
   ```

4. ユーザー承認後に本実行する（`--dry-run` を外す）。

   ```bash
   python3 skills/corporate-training-course-builder/scripts/build_exercise_gws_docs.py \
       --session-dir '講座/COURSE/01-セッション名' \
       --drive-folder-id 1AbCxyz...
   ```

5. 出力 JSON の `created[].url` を確認し、ブラウザで開いて、タブ数・各タブの中身・ドキュメント本文が正しいか確認する。

## 命名規則

- スプレッドシート: 既定で `<回フォルダ名>_演習データ`（`--sheet-title` で上書き可）。1回1ファイル。
- タブ名: CSV のファイル名（拡張子なし）。Sheets で使えない文字 `[ ] * / \ ? :` は空白に置換し、100文字に切り詰める。重複時は `_2`, `_3` を付ける。
- ドキュメント: ワークシートのファイル名（拡張子なし）。

## 失敗時の扱い

- gws が未認証・権限不足の場合は、エラーをそのまま報告し、ローカル CSV/Doc 代替で勝手に「完了」にしない。
- Drive フォルダ名が重複していて一意に決められない場合はエラーになる。`--drive-folder-id` でフォルダを直接指定する。
- CSV もワークシートも無い回では何も作らずエラー報告する（空データで既存成果物を上書きしない）。
- 本実行が途中で失敗した場合は、どこまで作成・移動できたか（出力 JSON）を残し、重複作成を避けるため再実行前に Drive を確認する。

## スクリプトがやらないこと

- 既存スプレッドシート/ドキュメントの上書き・マージはしない（毎回新規作成）。同じ回で作り直すときは、Drive 上の古いファイルの扱いをユーザーに確認する。
- 演習データの内容設計・修正はしない。データの中身は `session-specific-exercise-data-workflow.md` に従ってローカルで先に整える。
