# 04-MCPで社内ツールとデータをつなぐ設計 画像生成プロンプト（提出用・高密度版）

対象講座: AIエージェント実装・連携設計アカデミー
対象セッション: 04-MCPで社内ツールとデータをつなぐ設計
対象スライド: S01-S40
採用テンプレート: `isometric-corporate-clean`
密度基準: Google Workspace講座および直近サンプルS02「導入判断キャンバス」と同等。固有名詞・数値・事例は `skills/corporate-training-course-builder/references/concrete-tools-mcp-case-bank.md` に合わせる。

## 共通ルール

- `Course`、`Session`、フォルダ名は生成時の文脈情報であり、画像内の見出しやヘッダーとして描かない。画像内に出す文字は各スライドの `SOURCE TEXT` を正とする。
- GPT image 2 / built-in `image_gen` で1枚まるごとの完成ラスター画像として生成する。SVG、HTML/CSS、canvas、ブラウザスクリーンショット、ローカル変換、後載せテキスト合成は使わない。
- 各スライドは、タイトル、So What型ヘッドライン、3〜6個の内容カードまたは表・フロー・チェックリスト、架空事例、成果物、確認観点を入れる。
- サービス名やMCP名は目的とセットでテキストカードにする。実在ロゴ・実在UIは参照素材がある場合だけ使い、参照素材なしでは描かせない。
- 会社名は架空企業のみ。顧客名、社員名、メールアドレス、価格、連絡先、契約情報、APIキーは入れない。

## 具体サービス・MCP候補（この回で使う正本）

| 候補 | 業務での使いどころ |
| --- | --- |
| GitHub MCP | Issue・PR・リポジトリ参照を改善依頼やレビュー導線へつなぐ |
| Slack MCP | 未対応メンション抽出や日次通知の下書きを人の承認付きで扱う |
| Notion MCP | 議事録や社内Wikiを検索し、タスク化や更新案に使う |
| Google Drive MCP | 最新版資料の検索や参照範囲確認に使う。権限と共有範囲を先に確認する |

導入時の注意: MCP、コネクタ、サービスの提供状況、利用条件、料金、管理者設定、データ保持、監査ログ、商用利用条件は、必ず最新の公式情報で確認する。
