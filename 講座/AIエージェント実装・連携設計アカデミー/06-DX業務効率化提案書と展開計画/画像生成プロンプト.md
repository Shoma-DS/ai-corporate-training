# 06-DX業務効率化提案書と展開計画 画像生成プロンプト（提出用・高密度版）

対象講座: AIエージェント実装・連携設計アカデミー
対象セッション: 06-DX業務効率化提案書と展開計画
対象スライド: S01-S40
採用テンプレート: `isometric-corporate-clean`
密度基準: Google Workspace講座および直近サンプルS02「導入判断キャンバス」と同等。固有名詞・数値・事例は `skills/corporate-training-course-builder/references/concrete-tools-mcp-case-bank.md` に合わせる。

## 共通ルール

- `Course`、`Session`、フォルダ名は生成時の文脈情報であり、画像内の見出しやヘッダーとして描かない。画像内に出す文字は各スライドの `SOURCE TEXT` を正とする。
- GPT image 2 / built-in `image_gen` で1枚まるごとの完成ラスター画像として生成する。SVG、HTML/CSS、canvas、ブラウザスクリーンショット、ローカル変換、後載せテキスト合成は使わない。
- 各スライドは、タイトル、So What型ヘッドライン、3〜6個の内容カードまたは表・フロー・チェックリスト、架空事例、成果物、確認観点を入れる。
- サービス名やMCP名は目的とセットでテキストカードにする。実在ロゴ・実在UIは参照素材がある場合だけ使い、参照素材なしでは描かせない。
- 会社名は架空企業のみ。顧客名、社員名、メールアドレス、価格、連絡先、契約情報、APIキーは入れない。

## 具体サービス・MCP候補（この回で使う正本）

| 候補 | 業務での使いどころ |
| --- | --- |
| Codex | 第1〜5回の成果物を提案書の構成と根拠へ整理する |
| Notion MCP | 社内Wikiや議事録の根拠整理、タスク化の候補として扱う |
| Slack MCP | 現場通知や試行時の問い合わせ導線を設計する |
| GitHub MCP | 変更履歴、Issue、レビュー結果を提案の証跡にする |

導入時の注意: MCP、コネクタ、サービスの提供状況、利用条件、料金、管理者設定、データ保持、監査ログ、商用利用条件は、必ず最新の公式情報で確認する。

```text
Use case: productivity-visual / infographic-diagram. Create one complete 16:9 Japanese corporate training slide as a final raster image. Clean white background, navy headline, blue/teal/mint accents, thin light-gray card borders, subtle soft shadows, crisp isometric corporate infographic style. Make the slide understandable without speaker notes: include the title, So What headline, structured content cards/table/process/checklist, a fictional numeric business case, learner output or exercise band near the bottom, and risk/review notes. Do not render real brand logos or real UI unless reference assets are provided. Keep Japanese text short, readable, and faithful to SOURCE TEXT.
```

```text
Avoid incorrect Japanese, mojibake, unreadable tiny text, placeholder text, blank editable areas, dashed logo boxes, fake UI, invented real logos, real company names, real personal data, email addresses, phone numbers, pricing, contact information, credentials, QR codes, watermarks, dark backgrounds, decorative gradient blobs, comic style, messy layout, or any appearance of SVG/HTML/CSS/canvas/browser screenshot/local overlay output.
```
