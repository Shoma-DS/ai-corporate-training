# 第6回 スライド構成案: DX業務効率化提案書と展開計画

対象講座: AIエージェント実装・連携設計アカデミー

## 設計方針

- 採用テンプレート: `isometric-corporate-clean`
- 第1回から第5回の成果物を、導入判断者に説明できるDX業務効率化提案書へ統合する。
- 便利さの紹介ではなく、課題、効果仮説、検証結果、リスク、運用、展開計画をそろえる。
- 録画eラーニング前提で、一時停止ワーク、自己レビュー、講師記入例の比較で完成度を上げる。
- 数値は演習上の仮定として扱い、成果保証、価格、申込先、助成率など変わり得る情報は固定しない。

### S01 DX業務効率化提案書と展開計画

**ヘッドライン:** これまでの成果物を、効果仮説・リスク・運用計画を含む導入提案へ統合する
**テンプレートID:** `isometric-corporate-clean`
**図解パターン:** `scene-environment`
**素材・画面:** 図解
**主な内容:** 提案書、効果仮説、運用計画、展開ロードマップ
**推奨レイアウト:** 右側に小さな業務場面、左側にタイトル・ヘッドライン・3枚の要点カードを置く。実在UIやロゴは描かない。
**参照素材・スクリーンショット:** 生成図解を基本にする。実在ロゴ・実在UIは参照素材がある場合のみ使い、参照素材なしではサービス名を文字カードで示す。

**内容ブロック:**
- 判断すること
  - 提案書、効果仮説、運用計画、展開ロードマップを、業務課題、入力情報、AI/ツールの役割、人の確認、成果物、停止条件の観点で読む。
  - このスライドは `DX業務効率化提案書、KPI効果仮説、導入ロードマップ、次アクション一覧` へ接続する。
- 具体サービス・MCP候補
  - Codex, Notion MCP, Slack MCP, GitHub MCP を目的とセットで扱う。提供状況、利用条件、料金、管理者設定は導入時に最新の公式情報を確認する。
  - 実在ロゴや実在画面は想像生成せず、参照素材がない場合は文字カードで示す。
- 数値付き架空事例
  - みどり商事（卸売業・従業員60名）の問い合わせ対応と月次売上報告を題材に、削減時間は演習上の仮説として扱い、試行後にログで見直す提案にする。
  - 数値は演習上の例または想定例であり、成果保証ではない。
- 受講者アクション・確認観点
  - 提案書、効果仮説、運用計画、展開ロードマップを自部署の業務に置き換え、入力、処理、出力、人の確認、ログ/保存先、停止条件を書けるか確認する。

  - リスク確認: 実データ、顧客情報、社員情報、価格、連絡先、契約情報、認証情報、未公開資料は使わない。

**成果物・確認観点:** DX業務効率化提案書、KPI効果仮説、導入ロードマップ、次アクション一覧へ反映できる形で、成果物名、確認者、レビュー基準、停止条件を明記する。
