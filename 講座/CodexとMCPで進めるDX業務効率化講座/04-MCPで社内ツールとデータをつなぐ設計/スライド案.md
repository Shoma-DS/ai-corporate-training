# 第4回 スライド構成案: MCPで社内ツールとデータをつなぐ設計

対象講座: CodexとMCPで進めるDX業務効率化講座

## 設計方針

- 採用テンプレート: `isometric-corporate-clean`
- 120分の録画eラーニングとして、MCPを「AIエージェントの安全な接続設計」として扱う。
- MCPの技術用語は、Resources、Prompts、Tools、Host/Client/Server、認証、承認、ログに絞って、業務設計へ落とす。
- 実演はダミー設定と棚卸し表で行う。実在SaaS、実トークン、社内URL、顧客データは扱わない。

## 120分の時間配分

| ブロック | 時間 | スライド範囲 | ねらい |
| --- | ---: | --- | --- |
| MCPの全体像 | 20分 | S01-S08 | MCPの役割と第3回プロトタイプとの接続を理解する |
| Resources/Prompts/Tools設計 | 35分 | S09-S22 | 参照情報、定型手順、実行操作を分ける |
| 認証・承認・設定ドラフト | 25分 | S23-S30 | STDIO/HTTP、認証、承認モード、設定の扱いを整理する |
| 接続しない判断とリスク | 25分 | S31-S36 | 権限過多、外部送信、削除、ログ不足を避ける |
| ワークと第5回接続 | 15分 | S37-S40 | MCP連携設計書を仕上げ、運用設計へ渡す |
