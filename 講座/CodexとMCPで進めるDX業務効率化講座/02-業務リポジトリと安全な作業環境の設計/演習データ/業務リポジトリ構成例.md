# 業務リポジトリ構成例

```text
business-dx-prototype/
  AGENTS.md
  README.md
  docs/
    requirements.md
    workflow_as_is_to_be.md
  data/
    sample/
      inquiries.csv
      weekly_report_source.csv
  scripts/
    README.md
  outputs/
    README.md
  tests/
    README.md
  prompts/
    codex_task_brief.md
  runbooks/
    operation_runbook.md
    recovery_steps.md
```

## ポイント

- `data/sample/` はダミーデータだけにする。
- 実在の顧客情報、社員情報、契約情報、認証キーは置かない。
- `runbooks/` には運用と復旧手順を残す。
