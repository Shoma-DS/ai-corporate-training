# レベル3対応表

講座名: CodexとMCPで進めるDX業務効率化講座

## レベル3相当の到達状態

受講者が、CodexやMCPの使い方を知っているだけでなく、自社・自部署の業務課題を整理し、AIエージェント活用の要件を定義し、プロトタイプを検証し、リスクと運用を踏まえて導入提案できる状態を目指す。

| レベル3観点 | 講座で扱う内容 | 対応する成果物 |
| --- | --- | --- |
| 業務課題を自分で整理できる | 業務棚卸し、As-Is/To-Be、Gap分析、改善優先順位 | 業務棚卸しシート、改善テーマ選定メモ |
| 要件定義ができる | 目的、対象業務、入力、出力、人の確認、禁止事項、検証方法 | Codexタスクブリーフ、要件定義メモ |
| 解決策を設計できる | Codex活用、リポジトリ構成、ダミーデータ、MCP Resources/Prompts/Tools設計 | リポジトリ設計メモ、MCP連携設計書 |
| プロトタイプを作り検証できる | サンプルCSV処理、レポート生成、問い合わせ分類補助、テスト、レビュー | 業務効率化プロトタイプ、テスト結果 |
| リスクと運用を設計できる | 情報管理、権限、承認、ログ、プロンプトインジェクション、誤実行、手動復旧 | 運用設計書、リスクチェックリスト、復旧手順 |
| 効果を説明し提案できる | KPI、削減時間、業務品質、属人化解消、ロードマップ、教育、横展開 | DX業務効率化提案書、導入ロードマップ |

## ツール先行に見せないための表現方針

- 悪い表現: Codexの使い方を学ぶ、MCPサーバーを設定する、ツール操作を覚える
- 推奨表現: 業務課題をAIエージェントに渡せる要件へ分解する、MCP連携の権限と運用を設計する、プロトタイプを検証し導入判断につなげる

## 成果物の評価ルーブリック

| 評価項目 | 不十分 | 標準 | レベル3相当 |
| --- | --- | --- | --- |
| 課題整理 | 困りごとの列挙に留まる | 業務手順と課題が分かれている | 原因、影響、改善目的、KPIまで整理されている |
| Codex依頼 | 「作って」とだけ依頼する | 目的と出力形式がある | 対象ファイル、制約、禁止事項、テスト、完成条件がある |
| MCP設計 | つなぎたいツール名だけを書く | Resources/Toolsを分けている | 認証、承認、ログ、停止手順、接続しない判断まである |
| 検証 | 動けば完了にする | サンプルデータで確認する | 正常系、異常系、権限、出力レビュー、復旧まで確認する |
| 提案 | 作ったものを紹介する | 効果とリスクを説明する | 導入判断者向けに効果、リスク、運用、ロードマップを説明できる |
