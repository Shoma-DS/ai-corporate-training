# 第1回 演習ガイド

## 演習の目的

Codexで実装を始める前に、業務課題、改善テーマ、依頼条件を整理する。

## 手順

1. `演習データ/業務棚卸しサンプル.csv` を確認する。
2. 自社または共通ケースから、改善したい業務を3つ書く。
3. 1つを選び、As-Is/To-Beを5から7ステップで書く。
4. `演習データ/Codexタスクブリーフ例.md` を参考に、Codexタスクブリーフを書く。
5. `演習データ/改善テーマ評価サンプル.csv` を参考に、最初に扱うテーマをA/B/Cで分類する。

## 注意

- 実在の個人名、会社名、顧客名、メールアドレス、電話番号、契約番号は書かない。
- 認証キー、OAuth認証情報、社内URL、未公開資料名は書かない。
- 最初のテーマは小さくする。1つの入力から1つの出力を作れる範囲が望ましい。
