# 第4回 スライド構成案: Claude Codeで保守・レビュー・手順化を進める

対象講座: 事例で学ぶ中小企業のAIエージェント導入ワークショップ
スライド枚数: 40枚 / 120分
テンプレートID: `isometric-corporate-clean`（全スライド共通）
密度基準: `Google Workspace・GASで進めるAI業務効率化-DX実践講座` と同等の情報量。タイトルだけ・同一文型の繰り返し・抽象論だけのスライドは未完成扱い。

## 設計方針

- 目的: 既存ファイルや手順を読み解き、変更前の理解、レビュー、手順化、承認範囲の分離を行う。
- 架空ケース: 架空の管理部が、古い集計手順、簡易スクリプト、引き継ぎメモを安全に読み直す。公開資料には実在企業・顧客・社員・契約情報を入れない。
- 成果物: Claude Code作業依頼書、レビュー観点表、手順化メモ。各スライドはこの成果物のどこに効くかを明示する。
- 使用データ: 演習データ/事例カードサンプル.csv、演習データ/導入判断メモ.md。演習ではダミーデータのみを扱い、APIキー、Drive URL、個人情報、価格、連絡先は保存しない。
- 差別化軸: 変更を急がず、まず理解メモ・質問・差分説明を出させてから人が判断する。
- 録画eラーニング想定のため、演習は「動画を一時停止して記入」形式で指示する。

### S01 Claude Codeは既存資産を読み解き保守する導入に向く
**ヘッドライン:** Claude Codeは既存資産を読む、説明する、レビューする、手順化する場面で保守性を高める

**内容ブロック①：章内の位置づけ**
- 対象ブロック: 導入とClaude Codeの位置づけ（S01-S08 / 18分）
- このスライドで決めること: 「Claude Codeは既存資産を読み解き保守する導入に向く」を、Claude Codeで保守・レビュー・手順化を進めるの実務判断へ落とし込む。
- 後続成果物への接続: 第5回の利用ルール・承認フローへ渡す運用リスクと確認項目。

**内容ブロック②：Claude Code依頼書・レビュー表**
| 観点 | 具体内容 | 判断・確認ポイント |
|---|---|---|
| 読み取り | 既存手順、簡易スクリプト、引き継ぎメモを説明させる | 変更前に理解メモを出す |

| 手順化 | 現場担当者が読める手順へ変換する | 担当・頻度・停止条件を入れる |

**内容ブロック③：講座固有の具体例**
- このスライドは「Claude Code依頼書・レビュー表」として配置し、説明なしでも判断軸が読めるようにする。
- 演習では 演習データ/事例カードサンプル.csv、演習データ/導入判断メモ.md の架空データだけを使い、実在顧客名・社員名・契約情報は扱わない。

- ツール名の優劣ではなく、業務の入力・変更リスク・検証可能性から使い分ける。

**内容ブロック④：受講者の記入・確認ポイント**
- ワークシート記入欄: Claude Codeは既存資産を読み解き保守する導入に向くに関する「対象業務」「入力情報」「AIに任せる処理」「人の確認」「対象外」を1行で書く。
- 自己レビュー: 説明がツール名だけになっていないか、業務課題・成果物・確認者・停止条件が入っているかを確認する。
- 講師記入例との比較: 抽象語を削り、実務で明日試せる粒度へ直す。

- **図解パターン:** review-rubric
- **テンプレートID:** isometric-corporate-clean
