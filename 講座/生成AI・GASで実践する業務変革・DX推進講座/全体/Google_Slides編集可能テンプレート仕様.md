# Google Slides編集可能テンプレート仕様

対象講座: 生成AI・GASで実践する業務変革・DX推進講座

## 目的

マナビDX再申請用のテキスト教材は、全面画像だけで本文を固定せず、Google Slides上で講座名、テキスト名、セクション名、スライド番号、見出し、本文、表、チェックリストを編集可能なテキストとして配置する。

既存のスライド案が持つ情報量は維持する。修正対象は、密度を下げることではなく、審査者が約40枚の教材を同じ読み方で追えるように構成を整えることである。

## 共通ページ仕様

| 項目 | 仕様 |
| --- | --- |
| ページ比率 | 16:9 |
| 背景 | 白またはごく薄いブルーグレー |
| 基調色 | ネイビー、ブルー、ティール、ミント、薄いグレー |
| 書体 | Google Slides標準で再現しやすいNoto Sans JP系またはArial系 |
| 本文 | Google Slidesのテキストボックスで配置 |
| 表・チェックリスト | Google Slidesの図形、罫線、テキストボックスで配置 |
| 図解 | 別途生成した図解パーツ、公式素材、またはスクリーンショットを配置 |
| 禁止 | 全面1枚画像だけで本文、表、講座名、S番号を固定すること |

## 全スライド共通ヘッダー

全ページで同じ位置に置く。

| 位置 | 要素 | 表示例 | 編集可否 |
| --- | --- | --- | --- |
| 左上 | 講座名 | 生成AI・GASで実践する業務変革・DX推進講座 | 編集可能テキスト |
| 左上2行目 | セッション/テキスト名 | 第3回 GASによる業務プロセス自動化 | 編集可能テキスト |
| 右上 | スライド番号 | S08 | 編集可能テキスト |
| 右上2行目 | セクション名 | 2. GASの基本構造 | 編集可能テキスト |

ヘッダーの目的は、審査者がページ単体を見ても「どの講座の、どの回の、どの位置の教材か」を判断できるようにすることである。表紙、セクション扉、まとめスライドも例外にしない。

## 本文レイアウト

| 領域 | 役割 | 密度基準 |
| --- | --- | --- |
| タイトル | 1行でスライドの論点を示す | 単なる名詞だけにしない |
| ヘッドライン | So What型の結論文 | 1文で判断・効果・理由が分かる |
| 本文ブロック | カード、表、プロセス、比較、チェックリスト | 3〜6ブロックを原則にする |
| 図解/素材 | 概念図、画面、公式素材、補助ビジュアル | 本文を置き換えず理解を補助する |
| 下部帯 | 演習、成果物、確認観点、リスク | 受講者が何を作り何を確認するかを示す |

## セクション設計

各回は、120分の時間配分表に合わせて3〜6個のセクションに分ける。セクション名はスライド案の時間配分と一致させ、Google Slidesの右上ヘッダーに常時表示する。

| セッション | 標準セクション例 |
| --- | --- |
| 第1回 | 1. 導入と成果物 / 2. 業務DX動向 / 3. 業務課題の可視化 / 4. As-Is/To-Be設計 / 5. 改善候補選定 / 6. 演習とまとめ |
| 第2回 | 1. 導入と成果物 / 2. 入力設計 / 3. 台帳設計 / 4. 権限・保存設計 / 5. 演習とまとめ |
| 第3回 | 1. 導入とGASの位置づけ / 2. GASの基本構造 / 3. SpreadsheetApp処理 / 4. ログ・例外・テスト / 5. 演習とまとめ |
| 第4回 | 1. 導入とAIの役割 / 2. Gem/Geminiの使い分け / 3. 分類・要約・下書き / 4. 人による確認 / 5. 演習とまとめ |
| 第5回 | 1. 導入と要件定義 / 2. 自動化範囲設計 / 3. 運用・権限・ログ / 4. テスト・復旧 / 5. 演習とまとめ |
| 第6回 | 1. 導入と提案書ゴール / 2. KPIと効果試算 / 3. 技術構成と運用計画 / 4. リスク対策 / 5. 発表レビューとまとめ |

## 図解パーツの扱い

画像生成プロンプトは、完成スライド全面ではなく、以下のような図解パーツを作るために使う。

| 図解パーツ | 使いどころ | 画像内テキスト |
| --- | --- | --- |
| プロセス図 | 入力→処理→出力→確認→改善 | 短いラベルのみ |
| Before/After図 | 手作業と改善後の対比 | 3〜5語程度の短いラベル |
| 2x2マトリクス | 優先順位、リスク、難易度 | 軸名と象限名 |
| アーキテクチャ図 | Forms/Sheets/GAS/Gemini/人の役割 | ツール名は目的とセットで短く |
| ガバナンス図 | 権限、ログ、個人情報、確認 | 注意ラベルのみ |
| 表紙/章扉ビジュアル | セッション開始・区切り | 講座名やS番号はGoogle Slides側で置く |

図解パーツは、スライド中央または右側に置き、本文カード・表・チェックリストはGoogle Slides上の編集可能テキストとして残す。

## スライド種別

| 種別 | 用途 | 必須要素 |
| --- | --- | --- |
| 表紙 | 各回の開始 | 共通ヘッダー、セッションタイトル、今日の成果物、受講前確認 |
| セクション扉 | 構成の区切り | 共通ヘッダー、セクション名、これから扱う論点、次の成果物への接続 |
| 解説 | 概念、判断基準、業務例 | ヘッドライン、3〜6ブロック、必要に応じて図解パーツ |
| 操作/実演前 | 画面共有へ移る直前 | 見る画面、使うデータ、作業手順、確認ポイント |
| 演習 | 受講者の一時停止ワーク | 作業手順、所要時間、成果物、自己チェック |
| まとめ | 各回の終了 | 本日の成果物、次回につなぐ準備、リスク確認 |

## 再申請前チェック

- 全スライドに講座名、セッション/テキスト名、S番号、セクション名がある。
- 講座概要、パンフレット、詳細シラバス、Google Slidesのセッション名が一致している。
- S番号が欠番・重複していない。
- 別セッション名や別セッションの成果物が本文中に紛れ込んでいない。
- 各回S02の「今日の2時間で作るもの」と、パンフレット/詳細シラバスの成果物名が一致している。
- セクション名が時間配分表と一致している。
- 全面画像だけで本文を固定したページがない。
- 図解パーツは本文の補助であり、審査に必要な説明は編集可能テキストとして読める。
- 現在のスライド案と同等の情報量、具体例、成果物、確認観点を保っている。
