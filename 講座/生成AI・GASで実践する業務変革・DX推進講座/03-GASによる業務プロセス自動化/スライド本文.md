## S02. 今日の2時間で作るもの

- 未請求・未対応行を抽出する自動処理スクリプト
- 抽出結果を書き出す別シート
- 処理件数とエラーを残すログシート
- 自分の業務に読み替えるためのメモ

## S03. 第3回の流れ

- GASの位置づけを確認する
- JavaScriptの最小構文を読む
- SpreadsheetAppで表を読み書きする
- 抽出・転記・重複チェック・ログ追記を実装する
- カスタムメニューとテスト観点を整理する

## S04. 第1回・第2回のつながり

- 第1回: 自動化したい業務を選ぶ
- 第2回: フォームと台帳を設計する
- 第3回: 台帳に対してGAS処理を実装する
- 台帳が整っているほど、自動化は安定する

## S05. GASは表作業を減らすための実務道具

- 毎回同じフィルタ・コピー・貼り付けを減らす
- 重複や未対応の見落としを減らす
- 処理結果をログに残して属人化を減らす
- 人は例外判断と確認に集中する

## S06. GASでやること、やらないこと

| 領域 | 担当 |
| --- | --- |
| 決まった抽出・転記・集計 | GAS |
| 下書き・分類・要約 | Gemini/Gem |
| 承認・例外判断・外部送信 | 人 |
| ログ確認・改善 | 人 + GAS |

## S07. Apps Scriptの基本画面

- エディタで関数を書く
- 実行ボタンで試す
- 初回実行では権限を確認する
- ログで処理結果を確認する
- 安定したらメニューやトリガーを検討する

## S08. 情報管理と実行権限

- GASはGoogle Workspace上のデータに触れる
- 処理によって必要な権限が変わる
- 研修ではダミーデータのみを使う
- 実顧客、社員情報、契約情報、個人メールは扱わない

## S09. コードは「読む順番」を決める

1. 設定値を見る
2. 入力元を見る
3. 条件を見る
4. 出力先を見る
5. ログを見る

## S10. 変数と定数

- `const`: 途中で変えない値
- `let`: 処理中に変わる値
- シート名や条件は先頭にまとめる
- 読み替える場所を一箇所に集める

```javascript
const SOURCE_SHEET = '売上CSV';
const TARGET_SHEET = '未請求一覧';
```

## S11. 配列と2次元配列

- `getValues()` は表を配列として読む
- 1行目はヘッダー
- 2行目以降はデータ
- 1行ずつ見て、対象行だけ集める

## S12. オブジェクトでヘッダーを見る

- 列番号を固定すると、列追加や並び替えに弱い
- ヘッダー名から列番号を探す
- 業務側の列名とコードを対応させる

```javascript
const headerMap = Object.fromEntries(headers.map((name, index) => [name, index]));
```

## S13. 条件分岐

- ステータスが「未請求」なら抽出
- 「重複疑い」なら要確認
- 期限が今日以前ならリマインド候補
- 判断基準は表の列で確認できる形にする

## S14. ループ

- データ行を1件ずつ確認する
- 条件に合う行だけ出力用に集める
- 人が目視で見ていた確認を、同じ順番でコード化する

```javascript
for (const row of rows) {
  // 1行ずつ確認する
}
```

## S15. 日付と文字列の扱い

- 日付は表示とコード上の型が違う場合がある
- `120,000円` はそのままでは計算しにくい
- 空欄や表記ゆれは先に確認する
- 最初は文字列条件の抽出から始める

## S16. SpreadsheetAppで触る範囲

- Spreadsheet: ファイル全体
- Sheet: シートタブ
- Range: セル範囲
- Values: セルに入っている値

## S17. シートを取得する

- 今開いているスプレッドシートを取る
- シート名で対象シートを取る
- シート名が違うと処理できない

```javascript
const ss = SpreadsheetApp.getActive();
const sheet = ss.getSheetByName('売上CSV');
```

## S18. 範囲を取得する

- `getDataRange()` でデータ範囲をまとめて取る
- 小さな表では全体読み取りがわかりやすい
- 大量データでは範囲や処理分割を検討する

## S19. 読む・処理する・まとめて書く

1. `getValues()` でまとめて読む
2. 配列の中で条件処理する
3. `setValues()` でまとめて書く

## S20. `setValues()` の形を合わせる

- 書き込む範囲と配列の行数・列数を合わせる
- 対象行が0件でもヘッダーは出す
- 行ごとの列数をそろえる

## S21. ヘッダー行を基準にする

| 書き方 | 特徴 |
| --- | --- |
| 列番号で指定 | 短いが列変更に弱い |
| ヘッダー名で指定 | 少し長いが読み替えやすい |

## S22. 画面共有: まず読み取ってログに出す

- `売上CSV` シートを読み取る
- ヘッダーとデータ件数をログに出す
- まだシートは書き換えない
- 読み取り結果を確認してから次へ進む

## S23. GASで更新する列、人が更新する列

- GAS: チェック結果、処理ログ、転記済み、通知日時
- 人: 承認、例外判断、最終確認、外部送信
- 原本に近い列はなるべく直接上書きしない

## S24. Excel的処理をGASに置き換える

| 手作業 | GAS処理 |
| --- | --- |
| フィルタ | 条件抽出 |
| コピー貼り付け | 別シート転記 |
| 件数確認 | 集計 |
| 目視確認 | 重複チェック |
| メモ | ログ追記 |

## S25. 対象行を抽出する

- 未請求
- 未対応
- 確認待ち
- 期限超過
- 担当者未設定

条件は列名と値で表現する。

## S26. 別シートへ転記する

- 元データを直接消さない
- 出力先シートを分ける
- 抽出結果にチェック結果列を追加する
- 後から確認できる状態にする

## S27. 集計表を作る

- 担当者別件数
- カテゴリ別件数
- ステータス別件数
- 関数・ピボットで足りる場合はGASにしない

## S28. 重複チェック

- 伝票番号
- 受付番号
- 申請ID
- メールアドレス
- 重複は削除せず、まず「要確認」にする

## S29. ステータスを更新する

- 転記済み
- 処理済み
- 要確認
- 通知済み

まずは出力とログで確認し、安定後に元シート更新を追加する。

## S30. ログを残す

| ログ項目 | 内容 |
| --- | --- |
| 実行日時 | いつ実行したか |
| 処理名 | 何を実行したか |
| 抽出件数 | 何件処理したか |
| 重複件数 | 何件確認が必要か |
| 結果 | 完了、エラー、0件 |

## S31. 失敗しても戻れる設計

- 回答原本を消さない
- 出力先を分ける
- ログを残す
- 0件でも正常終了する
- 重複は要確認にする

## S32. 画面共有: 未請求抽出・転記・ログ追記

- `売上CSV` から対象行を抽出
- `未請求一覧` へ書き出す
- `処理ログ` に結果を追記
- 元シートは消さない

## S33. 現場担当者が押せるカスタムメニュー

- Apps Scriptエディタを開かずに実行できる
- メニュー名は業務担当者に伝わる名前にする
- 最初は手動実行で結果を確認する

```javascript
function onOpen() {
  SpreadsheetApp.getUi()
    .createMenu('研修メニュー')
    .addItem('未請求を抽出', 'extractUnbilledRows')
    .addToUi();
}
```

## S34. 手動実行と自動実行を分ける

| 実行方法 | 向いている場面 |
| --- | --- |
| 手動メニュー | 初期運用、確認しながら実行 |
| 時間主導トリガー | 毎日・毎週の定期処理 |
| フォーム送信トリガー | 受付直後の処理 |

## S35. エラーは隠さず見える化する

- 必要な列がない
- シート名が違う
- 権限が足りない
- 対象行が0件
- 重複がある

エラーや注意はログで確認できるようにする。

## S36. 入力チェック

- 必須ヘッダーがあるか
- 空欄がないか
- 想定外ステータスがないか
- 止めるエラーか、要確認にする注意かを分ける

## S37. 対象行なし・重複ありの扱い

| 状態 | 扱い |
| --- | --- |
| 対象行なし | 0件として正常終了 |
| 重複あり | 要確認として出力 |
| 必須列なし | 処理を止める |
| 空欄あり | 業務ルールに応じて確認 |

## S38. 制限と通知

- GASには実行時間やサービス利用回数の制限がある
- メール通知は送信条件と頻度を決める
- 本番化前にGoogle公式情報を確認する
- 小さく始め、ログと復旧手順を用意する

## S39. 演習: 未請求行抽出と転記

- 入力元: `売上CSV`
- 条件: 対応状況が `未請求` または `重複疑い`
- 出力先: `未請求一覧`
- ログ: `処理ログ`

## S40. サンプルコードの全体構造

1. 設定
2. 入力
3. 検証
4. 処理
5. 出力
6. ログ

## S41. 読み替える5項目

- シート名
- ヘッダー名
- 抽出条件
- 出力先
- ログ

まずこの5項目だけを自分の業務に置き換える。

## S42. テスト観点

- 対象行あり
- 対象行なし
- 空欄あり
- 重複あり

期待結果を書いてから実行する。

## S43. 自分の業務に置き換える

| 業務 | 抽出条件の例 |
| --- | --- |
| 問い合わせ | ステータスが未対応 |
| 申請 | 承認期限が近い |
| 日報 | 未提出または課題あり |
| 期限管理 | 期限超過かつ未完了 |

- GASは決まった表処理を安定させる道具
- 入力、条件、出力、ログに分けて読む
- 原本を守り、出力先とログを分ける
- 次回はGem/Geminiで分類・要約・下書きを扱う
