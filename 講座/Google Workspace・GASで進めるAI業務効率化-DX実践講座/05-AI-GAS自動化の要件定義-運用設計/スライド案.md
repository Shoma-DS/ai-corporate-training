# 第5回 スライド案: AI/GAS自動化の要件定義・運用設計

| No. | スライドタイトル | 主な内容 |
| ---: | --- | --- |
| 5-1 | 自動化は作って終わりではない | 継続運用、保守、例外対応 |
| 5-2 | 要件定義で決めること | 対象業務、利用者、入力元、出力先、GAS処理、AI処理、GeminiのGoogle Workspace連携利用可否 |
| 5-3 | 自動化方式の選び方 | 関数、手動AI、Gem/Gemini、GeminiのGoogle Workspace連携、GAS、トリガー |
| 5-4 | フォーム送信・通知・帳票 | 受付番号、確認メール、担当者通知、Drive保存、Docs差し込み |
| 5-5 | 情報管理と権限 | 個人情報、顧客情報、AI入力禁止、共有範囲、編集権限 |
| 5-6 | GASの制限 | 実行時間、メール送信、トリガー、同時実行、大量処理 |
| 5-7 | ログとAI出力レビュー | 処理結果、分類結果、要確認、エラー内容 |
| 5-8 | テスト観点 | 正常系、空欄、重複、AI誤分類、権限ミス、通知漏れ |
| 5-9 | Gemini連携利用不可時の代替設計 | Sheets台帳、Gemへの貼り付け、GAS手動実行で止まらない運用にする |
| 5-10 | 定着運用 | 担当者、引き継ぎ、改善会議、更新ルール |
| 5-11 | 演習 | 運用設計書とリスクチェックリストを作る |

素材指示:

- ログシート、エラー通知、権限設定、テスト観点表は実画面または表形式で見せる。
- GeminiのGoogle Workspace連携を使う場合は、使える環境と使えない環境の代替手順を同じスライドで見せる。
