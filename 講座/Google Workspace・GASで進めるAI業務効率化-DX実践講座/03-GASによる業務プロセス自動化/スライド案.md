# 第3回 スライド構成案: GASでExcel的な処理を自動化する

対象講座: Google Workspace・GASで進めるAI業務効率化/DX実践講座。

## 設計方針

- 2時間講座として、詳細シラバスの第3回「GASでExcel的な処理を自動化する」に合わせる。
- GASを高度な開発技術としてではなく、繰り返しの表作業を減らすための実務道具として扱う。
- サンプルコードの丸暗記ではなく、入力元、条件、出力先、更新列、ログ出力を読み替える力を重視する。
- 第2回で作ったフォーム・台帳設計を前提に、SpreadsheetAppを中心に行取得、条件抽出、転記、集計、重複チェック、ステータス更新、ログ追記を扱う。
- Apps Scriptエディタ、実行ログ、承認画面、スプレッドシート画面は、ダミー環境のスクリーンショットまたは画面共有で見せる。
- 実在の顧客情報、社員情報、メール、契約情報は使わない。演習は `演習データ/` の架空データだけで成立させる。
- 公式ロゴは `素材/ロゴ/` の取得済み素材を使い、画像生成AIに実在ロゴや実在UIを描かせない。

## スライドに入れるコード断片

### 基本の読み取り

```javascript
function checkRows() {
  const sheet = SpreadsheetApp.getActive().getSheetByName('売上CSV');
  const values = sheet.getDataRange().getValues();
  const headers = values[0];
  const rows = values.slice(1);

  Logger.log(headers);
  Logger.log(`${rows.length}件のデータがあります`);
}
```

### ヘッダー名から列を探す

```javascript
function createHeaderMap(headers) {
  return Object.fromEntries(headers.map((name, index) => [name, index]));
}
```

### カスタムメニュー

```javascript
function onOpen() {
  SpreadsheetApp.getUi()
    .createMenu('研修メニュー')
    .addItem('未請求を抽出', 'extractUnbilledRows')
    .addToUi();
}
```

## 講師の注意

- 受講者がコードの細部に入りすぎたら、「今日は業務条件へ読み替える力が主目的」と戻す。
- 初回実行の承認画面では、権限の意味を説明する。何でも許可すればよい、という説明にしない。
- 回答原本シートを直接上書きしない。出力用シート、管理台帳、ログシートを分ける。
- エラーをゼロにする話ではなく、エラーが起きた時に誰が、どこを見て、どう戻すかを扱う。
- AIやGeminiにコード読解を頼む場合も、実データや社内固有情報を貼り付けず、出力は人が確認する。
