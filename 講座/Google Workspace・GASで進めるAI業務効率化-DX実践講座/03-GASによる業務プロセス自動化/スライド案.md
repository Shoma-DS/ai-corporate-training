# 第3回 スライド構成案（リビルド版）: GASによる業務プロセス自動化

対象講座: 生成AI・GASで実践する業務変革・DX推進講座。

## 設計方針

- 2時間講座として、第3回「GASでExcel的な処理を自動化する」に合わせる。
- GASを高度な開発技術としてではなく、繰り返しの表作業を減らすための「事務ロボット」として位置づける。
- サンプルコードの丸暗記ではなく、入力元・条件・出力先・更新列・ログ出力を読み替える力を重視する。
- 第2回で作ったフォーム・台帳設計を前提に、SpreadsheetAppを中心に行取得・条件抽出・転記・集計・重複チェック・ステータス更新・ログ追記を扱う。
- 実在の顧客情報・社員情報・メール・契約情報は使わない。演習は `演習データ/` の架空データだけで成立させる。
- 「今から○分取ります」は使わず「動画を一時停止して〜分取り組んでください」を使う（録画動画形式）。

### S05 GASは「表作業の事務ロボット」と覚える

### S06 GASでやること・やらないこと・Geminiに任せること

**ヘッドライン:** 処理の種類を「GAS・Gemini・人」に振り分けることで、自動化の範囲が安全に広がる

**内容ブロック①：役割分担表**
| 担当 | 向いている処理 | 例 |
|---|---|---|
| GAS | 条件が決まっている機械的処理 | 未対応行の抽出、ステータス更新、ログ追記 |
| Gemini/Gem | 文章の分類・要約・下書き | 問い合わせカテゴリ分け、返信案、日報要約 |
| 人（担当者） | 承認・例外判断・最終確認 | 「要確認」フラグ行の確認、対外送信の判断 |
| Sheets | データ保持・閲覧・共有 | 台帳・集計表・ログの閲覧 |

**内容ブロック②：判断の流れ（図）**
```
データ入力（Forms）
  ↓
台帳に蓄積（Sheets）
  ↓
機械的な仕分け・集計（GAS）
  ↓
曖昧な文章は整理（Gemini/Gem）
  ↓
確認・承認・送信（人）
```

**内容ブロック③：「自動化しすぎない」3つの判断基準**
- 承認が必要なアクション（メール送信・外部転送）はGASに任せない
- 例外パターンが多い処理はGASでなく人が判断する
- ログを残さない自動実行は設計しない

- **図解パターン:** module-block

### S12 ヘッダー名から列番号を探す（headerMap）

**ヘッドライン:** 列番号を直接書かず、ヘッダー名から探す設計にすることで、列の追加・並び替えでコードが壊れなくなる

**内容ブロック①：列番号を直接書く問題**
```javascript
// NG: 列の並びが変わったら全部直さないといけない
const status = row[3];   // B列が追加されると壊れる
```

**内容ブロック②：headerMapで解決する**
```javascript
// OK: ヘッダー名から列番号を自動的に探す
function createHeaderMap(headers) {
  return Object.fromEntries(
    headers.map((name, index) => [name, index])
  );
}

// 使い方
const colMap = createHeaderMap(headers);
const status = row[colMap['請求ステータス']];  // 列が移動しても壊れない
```

### S13 条件分岐 ─ 「この行だけ処理する」を書く

### S19 読む・処理する・まとめて書く ─ 3ステップの基本

**ヘッドライン:** 「全件読む→条件で絞る→まとめて書く」の3ステップを守ることで、処理が速く、上書きミスが起きにくくなる

**内容ブロック①：3ステップの全体構造**
```javascript
// Step 1: まとめて読む
const values = inputSheet.getDataRange().getValues();
const headers = values[0];
const rows    = values.slice(1);
const colMap  = createHeaderMap(headers);

// Step 2: 条件で絞る（メモリ内で処理）
const result = [];
for (const row of rows) {
  if (row[colMap['請求ステータス']].toString().trim() === '未請求') {
    result.push(row);
  }
}

// Step 3: まとめて書く（1回だけ）
if (result.length > 0) {
  outputSheet.getRange(2, 1, result.length, result[0].length)
             .setValues(result);
}
Logger.log(`未請求: ${result.length}件`);
```

### S26 別シートへ転記する

**ヘッドライン:** 抽出結果を専用の出力シートへ書き出すことで、元データを汚さずに「今日の未対応一覧」が自動で完成する

```javascript
function extractAndTransfer() {
  const ss          = SpreadsheetApp.getActive();
  const inputSheet  = ss.getSheetByName('問い合わせ台帳');
  const outputSheet = ss.getSheetByName('未対応一覧');
  if (!inputSheet || !outputSheet) {
    Logger.log('シートが見つかりません'); return;
  }

  const values  = inputSheet.getDataRange().getValues();
  const headers = values[0];
  const rows    = values.slice(1);
  const colMap  = createHeaderMap(headers);

  // 出力シートをヘッダー行以外クリア
  const lastRow = outputSheet.getLastRow();
  if (lastRow > 1) outputSheet.getRange(2, 1, lastRow - 1, headers.length).clearContent();

  // 抽出
  const result = rows.filter(row =>
    row[colMap['ステータス']].toString().trim() === '未対応'
  );

  // 書き込み
  if (result.length > 0) {
    outputSheet.getRange(2, 1, result.length, headers.length)
               .setValues(result);
  }
  Logger.log(`転記完了: ${result.length}件`);
}
```

### S28 重複チェック ─ Set で重複を検出する

**ヘッドライン:** `Set` を使うと伝票番号・受付番号・メールアドレスの重複を瞬時に検出でき、目視確認を自動化できる

**内容ブロック①：重複チェックの基本**
```javascript
function checkDuplicates() {
  const sheet  = SpreadsheetApp.getActive().getSheetByName('売上CSV');
  const values = sheet.getDataRange().getValues();
  const headers = values[0];
  const rows    = values.slice(1);
  const colMap  = createHeaderMap(headers);

  const seen       = new Set();
  const duplicates = [];

  for (const row of rows) {
    const key = row[colMap['受付番号']].toString().trim();
    if (seen.has(key)) {
      duplicates.push({ key, row });
      Logger.log(`重複: ${key}`);
    } else {
      seen.add(key);
    }
  }

  Logger.log(`重複: ${duplicates.length}件 / 全体: ${rows.length}件`);
}
```

### S29 ステータスを更新する ─ 元シートへ書き戻す

```javascript
function updateStatus() {
```

### S40 サンプルコードの全体構造 ─ 設計の読み方

**ヘッドライン:** コードを「設定→入力→検証→処理→出力→ログ」の6ブロックに分けて読むと、どこを変えればよいかがすぐわかる

**内容ブロック①：完全なサンプルコード（コメント付き）**
```javascript
// ─── 設定ブロック ─────────────────────────────────
const INPUT_SHEET  = '問い合わせ台帳';
const OUTPUT_SHEET = '未対応一覧';
const LOG_SHEET    = '処理ログ';
const STATUS_COL   = 'ステータス';
const TARGET_VAL   = '未対応';
// ────────────────────────────────────────────────

function extractAndTransfer() {
  const ss = SpreadsheetApp.getActive();

  // ─── 入力 ────────────────────────────────────────
  const inputSheet  = ss.getSheetByName(INPUT_SHEET);
  const outputSheet = ss.getSheetByName(OUTPUT_SHEET);
  const logSheet    = ss.getSheetByName(LOG_SHEET);

  // ─── 検証 ────────────────────────────────────────
  [inputSheet, outputSheet, logSheet].forEach((s, i) => {
    if (!s) throw new Error(`シートが見つかりません: インデックス${i}`);
  });

  const values  = inputSheet.getDataRange().getValues();
  const headers = values[0];
  const rows    = values.slice(1);
  const colMap  = createHeaderMap(headers);
  validateHeaders(headers, [STATUS_COL]);

  // ─── 処理 ────────────────────────────────────────
  const result = rows.filter(row =>
    row[colMap[STATUS_COL]].toString().trim() === TARGET_VAL
  );

  // ─── 出力 ────────────────────────────────────────
  clearOutputSheet(outputSheet);
  if (result.length > 0) {
    outputSheet.getRange(2, 1, result.length, headers.length)
               .setValues(result);
  }

  // ─── ログ ────────────────────────────────────────
  appendLog(logSheet, '未対応抽出', result.length, 0, '正常終了');
  Logger.log(`完了: ${result.length}件`);
}
```

**内容ブロック②：ブロック別の「変えてよい場所・変えてはいけない場所」**
| ブロック | 変えてよい | 変えてはいけない |
|---|---|---|
| 設定ブロック | シート名・列名・条件値 | ─ |
| 入力 | シート名の定数 | getDataRange / getValues の構造 |
| 検証 | チェックする列名 | エラー処理の仕組み |
| 処理 | filter の条件 | result変数への追加方式 |
| 出力 | 出力先シート・列数 | setValues の構造 |
| ログ | 処理名・備考 | appendLog の引数の順番 |

- **図解パターン:** code-structure-diagram（6ブロック色分け）
- **テンプレートID:** isometric-corporate-clean
- **スクリーンショット:** なし
