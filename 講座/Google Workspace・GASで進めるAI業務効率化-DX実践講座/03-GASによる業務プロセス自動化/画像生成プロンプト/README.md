# 第3回 画像生成プロンプト 共通ルール

対象講座: Google Workspace・GASで進めるAI業務効率化/DX実践講座。

対象スライド: S01-S44

採用テンプレート: `isometric-corporate-clean`

## 保存先

- プロンプト: `画像生成プロンプト/S01-S11.md` など、範囲別に保存する。
- 生成画像: `スライド画像/S01.png` から `スライド画像/S44.png` に保存する。

## 共通ベースプロンプト

```text
16:9、1920x1080の法人研修用スライド画像を作成。白背景、ネイビーとティールのアクセント、薄いグレーのカード罫線、控えめな影、清潔なアイソメトリック図解、実務的で読みやすいビジネス資料デザイン。日本語テキストは大きく正確に、ゴシック体風で配置。指定した日本語以外の文字は入れない。実在企業名、人物名、価格、連絡先、QRコード、透かしは入れない。
```

## 共通ネガティブプロンプト

```text
文字化け、誤字、余計な文字、小さすぎる文字、読めない日本語、長文の詰め込み、参照画像なしでのロゴ再現、参照画像なしでのUI再現、実在企業名、人物名、価格、連絡先、QRコード、透かし、写真風人物、暗い背景、派手なグラデーション、装飾過多、漫画風、手書き風、過度な3D、雑然としたレイアウトを避ける。
```

## サービス画面・ロゴの扱い

- Apps Script、Google Sheetsなどの実在ロゴや実在UIは、画像生成AIで記憶から再現しない。
- ロゴが必要なスライドは、`素材/ロゴ/` にある公式素材を後から配置する。
- 画面共有スライドは、実画面を生成せず、「画面共有で確認」という切替案内として作る。
- ダミー環境スクリーンショットを入れる場合は、生成画像側には空の素材配置枠だけを作る。

## 情報管理

- 実顧客、社員名、メールアドレス、契約情報、価格、連絡先、社内固有情報を入れない。
- サンプルの業務データは、`演習データ/` の架空データを前提にする。
- GASコードは教材用の短い断片だけにし、本番の秘密情報やIDは入れない。
