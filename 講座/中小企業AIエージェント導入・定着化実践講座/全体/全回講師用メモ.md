# 全回講師用メモ

講座名: 中小企業AIエージェント導入・定着化実践講座

## 講師の基本姿勢

- AIエージェントを万能化しない。導入判断、確認、停止、定着までを扱う。
- CodexとClaude Codeの優劣比較ではなく、業務課題に応じた使い分けとして説明する。
- 中小企業の現場で始めやすいように、1業務、1週間、ダミーデータ、戻せる範囲を繰り返す。
- 実在企業の成果保証や固有情報を語らず、架空ケースとして扱う。

## 例え

- AIエージェントは新入社員ではなく、作業環境と依頼書が必要な臨時メンバー。
- Codexは小さな改善物を作って検証する担当。
- Claude Codeは既存資産を読み解き、レビューし、手順化する担当。
- 導入判断キャンバスは、社内説明前の設計図。

## 各回の注意

| 回 | 注意点 |
| ---: | --- |
| 1 | 事例の成果だけを真似させず、導入条件を読ませる。 |
| 2 | ツールから考えず、業務棚卸しから使い分けへ進める。 |
| 3 | Codexで作る範囲を小さくし、検証条件を必ず入れる。 |
| 4 | Claude Codeを魔法の修正者にせず、読解・レビュー・手順化から始める。 |
| 5 | リスク説明で怖がらせず、止められる運用を設計する。 |
| 6 | 提案書は誇大な効果保証ではなく、試行結果と次の検証で説明する。 |
