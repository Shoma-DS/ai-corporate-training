# ワークフロープラン: AX・DXワークショップ講座 スライド全面リビルド

## Goal
合格済みスライド（第1回・第2回）を参考資料として、情報量を1スライドあたり増強した新スライド案・画像生成プロンプトを全面的に作り直す。

## Success criteria
- 第1回・第2回それぞれの `スライド案.md` が完成（各回40〜50枚）
- 各スライドに結論型ヘッドライン + 構造化コンテンツ（プロセス、比較表、チェックリスト等）
- 1スライドあたりの情報量が元スライドより増加（見出し+意味あるブロック3〜6個）
- テンプレートID `isometric-corporate-clean` を全スライドで統一指定
- 各スライドにDiagram Pattern指定あり
- 画像生成プロンプトは第1フェーズでは省略し、スライド案・講師台本に集中
- 公開リポジトリに載せてはいけない情報（価格・顧客情報・会社固有情報）を含まない

## Current context
- 合格済みPDF保存先: `非公開/法人研修Drive原本/AX・DXワークショップ講座/合格済みスライド/`
- 出力先: `講座/AX・DXワークショップ講座/`
- テンプレート: `isometric-corporate-clean`

## Constraints
- public repo のルール遵守
- 画像生成はCodexへ委譲（本ワークフローでは画像生成プロンプトを作成するが画像は生成しない）
- 合格済みスライドの構成を「参考」にするが、丸ごとコピーしない

## Risks
- スライド案が長くなりすぎて可読性が下がる → 各スライドのコンテンツブロックを5個以内に抑える
- 情報密度が上がりすぎて視覚的に重くなる → 各ブロックは箇条書き3行以内に収める

## Approval required
- 画像生成（Codex委譲）はユーザー承認後に実施

## Workflow artifact path
`.workflow/ax-dx-rebuild/`

## Work packets
1. [P01] ワークフロー設計・フォルダ作成 → 完了
2. [P02] 第1回スライド案作成（44枚・120分）→ 実行中
3. [P03] 第2回スライド案作成（50枚・120分）
4. [P04] 全体整合チェック・ファイル確認
5. [P05] 画像生成プロンプト作成（ユーザー承認後）

## Integration policy
スライド案完成後、タイトル・スライド番号・テンプレートID・図解パターンの一貫性を確認してから次フェーズへ進む。

## Verification
- スライド番号連番確認
- テンプレートID統一確認
- 各スライドのヘッドライン・コンテンツブロック数確認
- public-safety チェック
