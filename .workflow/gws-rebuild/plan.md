# ワークフロープラン: Google Workspace講座スライド全面リビルド

## Goal
全6回のスライド案を「AX・DXワークショップ講座」形式（So Whatヘッドライン・比較表・業種別事例・演習・スクリーンショット参照付き）で全面リビルドする。内容はGWS講座固有のものを維持し、情報量・構図・具体度・例の出し方だけを参考形式に合わせる。

## Reference format (AX・DXワークショップ形式)
### S01 スライドタイトル
**ヘッドライン:** So What型の1文（結論→根拠の順）

本文: 比較表 or 箇条書き3〜6ブロック（業種別具体例・数値・Before/After含む）

- **図解パターン:** [カタログのID]
- **スクリーンショット:** `スクリーンショット/ファイル名.png` （取得先URL）

## 担当割り当て
- Agent A: 第1回・第2回
- Agent B: 第3回・第4回
- Agent C: 第5回・第6回

## Template ID
isometric-corporate-clean（第1〜3回・第6回）
soft-isometric-corporate-warm（第4〜5回）

## 共通ルール
- 各スライドに必ず「ヘッドライン」（So What型結論）を設ける
- 表・比較・プロセス図・チェックリストのいずれかの構造を各スライドに入れる
- 業種別具体例（製造・サービス・小売・士業等）を各章に最低1つ入れる
- 演習スライドは「手順・成果物・確認ポイント・自己レビュー」を詳細記載
- スクリーンショットが必要なスライドにはURL記載 + curlでダウンロード
- 1スライドあたりのコンテンツブロック: 3〜6個（元の2〜3倍の情報量）
- public-safety: 会社固有情報・価格・連絡先は含めない
