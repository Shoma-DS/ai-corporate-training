# AX・DXワークショップ講座 レベル3設計 壁打ちメモ

## 目的

まなびDXの講座レベル3相当として、生成AI・RAG・データ活用を単なるツール操作ではなく、業務課題の整理、要件定義、解決策設計、企画提案まで独力で進められる研修にする。

## 現時点の良い骨格

- 6回、約12時間、eラーニング/LMS前提の形式は、リスキリングや助成金関連の説明に載せやすい。
- DX/AXの全体像、生成AI活用レイヤー、RAG、データサイエンス、最終提案書という流れは、基礎理解から実務アウトプットへ進む構成になっている。
- 最終成果物をAX/DX企画提案書に置いているため、レベル3の「独力で要求作業を遂行する」状態に接続しやすい。
- RAG設計、データ棚卸し、KPI、ROI、ロードマップを扱う方向性は、ビジネスアーキテクトとデータサイエンティストの橋渡しとして説明しやすい。

## レベル3に近づけるための補強ポイント

1. 各回の成果物を明確にする

現状は学習目標が中心なので、各回で作るものを固定する。例として、第1回は自部署DX/AX成熟度マップ、第2回は生成AI適用判断シート、第3回はRAG対象データ棚卸し表、第4回は分析テーマ選定メモ、第5回は技術選定・評価指標表、第6回は企画提案書。

2. 独力遂行の評価基準を入れる

「理解できる」だけでなく、「指定された業務課題に対して、課題、対象データ、技術方式、リスク、評価指標、導入手順を説明できる」など、採点可能な行動基準にする。

3. セキュリティと情報管理を成果物に組み込む

生成AI、RAG、社内ナレッジを扱うため、機密情報、著作権、ハルシネーション、権限管理、ログ確認、更新運用を最終提案書の必須項目にする。

4. データマネジメント観点を足す

RAGや分析の前提として、データ所有者、更新頻度、品質、形式、アクセス権、利用可否を整理する。レベル3では「使えそうなデータを挙げる」だけでなく、「使える状態にする条件」を判断できる必要がある。

5. 企画提案書を1枚で終わらせない

提出物は1枚サマリーに加えて、根拠資料を持たせる。最低限、As-Is/To-Be、課題優先度、対象業務フロー、データ棚卸し、技術選定理由、KPI、リスク、導入ロードマップを含める。

## 壁打ちしたい論点

- 対象ロールを「ビジネスアーキテクト寄り」にするか、「データサイエンティスト入門寄り」にするか。
- 受講者の前提を、生成AI基礎あり・Excel集計経験あり程度に置くか、業務改善リーダー経験ありまで求めるか。
- 最終課題を自社課題で作るか、共通ケースで作るか。助成金や審査を意識するなら、共通ケースで評価し、自社転用は任意課題にすると採点しやすい。
- RAGを「設計まで」にするか、Difyなどで簡易プロトタイプ作成まで入れるか。
- データサイエンス回を広く紹介するか、KPI設計と評価指標に寄せて実務提案の完成度を上げるか。

## 推奨する方向性

講座の軸は「AI・データ活用プロジェクトを企画できる業務側DX人材」に置くのがよい。ツール操作やモデル実装を深掘りしすぎるより、課題設定、データ棚卸し、要件定義、評価、運用までを一貫して設計できることを前面に出す。

その場合の講座名は、次のようにすると審査・営業資料の両方で伝わりやすい。

- AX・DX企画実践講座
- 生成AI・データ活用プロジェクト設計講座
- AI活用/DX推進リーダー育成ワークショップ
